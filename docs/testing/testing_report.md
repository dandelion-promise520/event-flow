# EventFlow - 系统功能测试报告

本测试报告详细记录了 EventFlow 活动与票务管理系统的质量保障工作。系统采用了 **自动化集成测试（白盒测试）** 与 **功能用例手工校验（黑盒测试）** 相结合的测试策略，对系统的核心业务逻辑进行了全面验证。

---

## 一、 测试概要说明

### 1. 测试环境
- **操作系统**：Windows 11 / macOS 14 / Linux Ubuntu 22.04
- **运行期环境**：Node.js v20.x + pnpm v9.x
- **底层数据库**：SQLite 3.x
- **前端渲染端**：Next.js 16.2 (App Router)

### 2. 测试框架与工具
- **自动化测试工具**：Node.js 原生测试运行器 (`node --test`) 与 `tsx` (TypeScript 执行器)
- **数据库还原工具**：Prisma Client + SQLite 事务回滚与本地模拟隔离

---

## 二、 单元与集成测试用例设计

本系统针对核心 API 与数据库交互逻辑编写了 12 个独立的自动化测试脚本，涵盖以下核心业务：

| 序号 | 测试脚本名称 | 测试类型 | 测试目标与验证项 |
| :--- | :--- | :---: | :--- |
| 1 | `auth.test.ts` | 接口测试 | 验证用户登录接口对“密码正确”、“密码错误”、“账号不存在”及参数缺失的响应处理。 |
| 2 | `categories.test.ts` | 数据库/API测试| 验证活动分类的拉取、分类名称唯一性校验，确保检索分类数据准确。 |
| 3 | `events.test.ts` | 接口集成测试 | 验证活动的增删改查。重点测试“活动开始时间早于当前时间”的边界防错逻辑。 |
| 4 | `tickets.test.ts` | 接口集成测试 | 模拟学生在线预约抢票。验证“门票超限抢购拦截”、“单人重复购票拦截”等核心业务。 |
| 5 | `admin-stats.test.ts` | 统计集成测试 | 模拟多维度活动及门票销售，验证管理员大屏中各指标（活动总数、售票总数、活跃率）的计算公式。 |
| 6 | `dashboard-analytics.test` | 分析集成测试 | 验证组织者仪表盘的购票趋势折线图、不同活动核销率的数值计算精度。 |
| 7 | `notifications.test.ts` | 消息集成测试 | 验证当用户订票或被核销时，系统自动插入消息流的准确性，以及已读标记。 |
| 8 | `reviews.test.ts` | 评价集成测试 | 验证已消费用户发表评价的流程，约束单人单活动只能评价一次的限制条件。 |
| 9 | `users-management.test.ts` | 后台管理测试 | 验证系统管理员对组织者账号、普通学生账号的查询、角色调整与封禁逻辑。 |

## 三、 系统手工功能测试记录（黑盒测试）

为模拟用户真实使用场景，对前端各交互界面进行了手工功能用例测试，用例清单如下：

| 测试模块 | 用例名称 | 预置条件 | 测试步骤说明 | 期望结果 | 测试结果 |
| :--- | :--- | :--- | :--- | :--- | :---: |
| **用户认证** | 正常注册与登录 | 无 | 1. 访问 `/login` 页面，输入新邮箱与密码。<br/>2. 点击注册后，使用该邮箱进行登录。 | 成功进入控制台，页面显示对应用户昵称。 | **PASS** |
| **活动浏览** | 分类检索活动 | 数据库包含不同分类活动 | 1. 进入首页，点击“文体比赛”分类标签。<br/>2. 观察活动卡片列表是否发生过滤。 | 页面仅展示分类为“文体比赛”的活动。 | **PASS** |
| **在线订票** | 预订免费活动门票 | 用户已登录，活动容量未满 | 1. 进入《2026年校园极客马拉松》详情页。<br/>2. 点击“立即预订”按钮。 | 提示“预订成功”，并生成以 `EVT-` 开头的12位电子核销码。 | **PASS** |
| **门票核销** | 组织者控制台核销 | 组织者登录，手持学生核销码 | 1. 登录计算机协会账号。<br/>2. 进入核销台，输入学生门票码 `TKT-XM-UNUSED`，点击确认。 | 界面提示“核销成功：持票人张小明”，门票状态变更为已使用。 | **PASS** |
| **通知推送** | 购票成功通知接收 | 用户抢票成功 | 1. 抢票完成后，点击右上角消息通知小铃铛。<br/>2. 查看通知列表。 | 收到标题为“购票成功通知”的系统消息。 | **PASS** |
| **重复抢票** | 重复预约拦截机制 | 用户已拥有一张该活动门票 | 1. 再次进入已预约的活动详情页。<br/>2. 点击“立即预订”按钮。 | 按钮不可点击，接口返回“您已报名该活动”。 | **PASS** |
